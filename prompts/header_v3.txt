3. ИНСТРУКЦИЯ ПО ИЕРАРХИЧЕСКОЙ НОРМАЛИЗАЦИИ
Разложи ВСЕ данные документа по следующим КЛЮЧЕВЫМ секциям:

* **document_info:**
    * `type`, `number`, `date_raw` (сырая дата), `date_iso` (YYYY-MM-DD), `place_of_issue`, `currency`.
* **parties:**
    * Объект, содержащий объекты для каждой стороны. Ключом объекта должен быть **normalized_key** роли (`supplier`, `buyer`, `payer` и т.д.).
    * Каждая сторона должна содержать: `role` (оригинальное название роли), `name` (название организации).
    * Вложенный объект `details` для всех реквизитов: `address`, `phone`, `email`, `tax_status`.
        * **Коды:** `edrpou`, `ipn`, `vat_id`.
        * **Банковские данные (ГЛУБОКАЯ ВЛОЖЕННОСТЬ):** `bank_account` должен содержать `iban`, `bank_name`, `mfo` и другие найденные реквизиты.
* **references:** Ссылки на договор, счет, заказ или другие документы-основания.
* **totals:**
    * Общие итоговые суммы (без НДС, НДС, с НДС, включая транспортные расходы, если они указаны в нетабличной части).
* **amounts_in_words:** Текстовое выражение сумм (прописью).
* **signatures:**
    * Информация о всех подписантах. Извлекай **напечатанные** имена и роли.
    * Используй `name`, `role`, `signature_present` (true/false), `stamp_present` (true/false).
* **other_fields (КРИТИЧНО - ДЛЯ МАКСИМАЛЬНОГО КАЧЕСТВА):**
    * **СЕКЦИЯ ДЛЯ СОХРАНЕНИЯ ВСЕХ ОСТАЛЬНЫХ НАПЕЧАТАННЫХ ДАННЫХ.**
    * Сюда помещай все поля и текстовые блоки, которые не вошли ни в одну из вышеперечисленных категорий.
    * Используй массив объектов вида `{ "label": "Сырой текст ярлыка", "value": "Значение" }`.
    * **НИЧЕГО НАПЕЧАТАННОГО НЕЛЬЗЯ ПОТЕРЯТЬ.**
